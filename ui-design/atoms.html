Spacings and sizes

Spacings (paddings, margins)

Spacing determines the neutral space between elements. It's controlled through margins and padding. Our spacing is determined by increments of 0.4. We start at 0.4, then increment by that value.

Spacing variables are defined as follow:

  • --space-xs: 0.4rem;
  • --space-s: 0.8rem;
  • --space-sm: 1.2rem;
  • --space-m: 1.6rem;
  • --space-ml: 2.0rem;
  • --space-l: 2.4rem;

The above values are css variables that can be applied in place of a "hardcoded" value, like so:

Do:

  • border-bottom: #a74500 solid var(--space-xs);
  • border-right: #a74500 solid var(--space-xs);

Don't:

  • border-bottom: #a74500 solid 8px;
  • border-right: #a74500 solid 8px;

Spacing and margins are better applied on the parent element of a item, to ensure that formatting is coming from the outside in. This is achieved by using the `grid` layout, as such:

display: grid;
grid-template-columns: auto;
gap: var(--space-m);

This formats content in a way that keeps spacing coherent and even accross all elements. It can be manipulated to add more columns, retaining a regular gap.

Sizes

Sizing is relative to the height and width of things (titles, buttons, colored elements...).

We apply the same unit grid to sizing than we did to spacing, meaning that our increment value is 0.4; the difference is that the base value is 16px (the industry standard).

All sizing MUST be defined using rem , which are a responsive unit based on the root font size (16px). This allows us to maintain consistent, harmonious and responsive sizing accross the app.

Sizing variables are defined as follow:

--xxl: 6rem
--xl: 5rem
--l: 4rem
--m: 3rem
--regular: 1rem
